---
title: Breakpoints
---

Breakpoints allow you to build responsive layouts by reacting to screen size changes.
There are 6 predefined breakpoint values, `xs`, `sm`, `md`, `lg`, `xl` and `default`.
Breakpoints are triggered once the screen width is greater than the specified width value.

<Example>
  <DataTable
    data={[
      { id: 1, name: 'default', device: 'Phones (portrait)', width: '0px' },
      { id: 2, name: 'xs', device: 'Phones (landscape)', width: '520px' },
      { id: 3, name: 'sm', device: 'Tablets (portrait)', width: '728px' },
      { id: 4, name: 'md', device: 'Tablets (landscape)', width: '1024px' },
      { id: 5, name: 'lg', device: 'Laptops', width: '1280px' },
      { id: 6, name: 'xl', device: 'Desktops', width: '1640px' },
    ]}
  >
    <DataColumn id="name" label="Size" />
    <DataColumn id="device" label="Device" />
    <DataColumn id="width" label="Width" />
  </DataTable>
</Example>

## Usage

Certain properties of layout components accept a `Responsive` object, which specifies what value
to apply at a given breakpoint.

<Example>
  <Heading
    size={{
      default: '1',
      xs: '2',
      sm: '3',
      md: '5',
      lg: '6',
      xl: '7',
    }}
  >
    {`This text will shrink/grow as the screen resizes.`}
  </Heading>
</Example>

```jsx
<Heading
  size={{
    default: '1',
    xs: '2',
    sm: '3',
    md: '5',
    lg: '7',
    xl: '9',
  }}
/>
```
